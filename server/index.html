Roostagram API Specification

Supports the following actions:

  • Get a photo
  • Take a photo
  • Delete a photo
  • View all photos

End Points:

GET /photos/:id

Returns data about a specific photo.

Example: /photos/1

Params:

id - String. photo id

Responses:

200 OK

Return data:

{
    "id": "{String}",
    "dataUri": "{String}"
}

Example:

{
    "id": "1",
    "dataUri": "data:image/jpeg:base64,/9j/34thgkj...hk37"
}

404 NOT FOUND

{ "error" : "Photo not found" }

POST /photos

Posts a new photo

Example: /photos

Sent data in post body:

String, dataUri of image. Example: "data:image/jpeg:base64,/9j/34thgkj...hk37"

Responses:

200 OK

Return data:

{
  "id": "{String}",
  "dataUri": "{String}"
}

Example:

{
  "id": "12",
  "dataUri": "data:image/jpeg:base64,/9j/34thgkj...hk37"
}

GET /photos

Gets all the available photos

Example: /photos

Params:

None

Responses:

200 OK

[
  { "id" : "{String}", "dataUri" : "{ String }" },
  { "id" : "{String}", "dataUri" : "{ String }" } // ...
]

Example:

[
  { "id" : "12", "dataUri" : "data:image/jpeg:base64,/9j/34thgkj...hk37"},
  { "id" : "13", "dataUri" : "data:image/jpeg:base64,/3jwwreh4thgj...hfe7"} // ...
]

DELETE /photos/:id

Deletes a photo

Example: /photos/13

Parameters:

id - photo id

Returned data

200 OK

{ "id" : "{String}" }

Example:

{ "id" : "13" }

404 NOT FOUND

{ "error" : "Photo not found" }
